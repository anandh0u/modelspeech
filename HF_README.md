# EmoMemory: Memory-Enabled Emotion Intelligence

**Powered by Cognee | Built for WeMakeDevs Hackathon 2025**

## 🎯 What is EmoMemory?

EmoMemory solves the fundamental problem of **stateless AI** by giving emotion intelligence systems a **permanent memory layer** powered by Cognee. Traditional emotion AI treats each interaction independently, losing valuable context about user emotional patterns.

## ✨ Key Features

- **🧠 Persistent Memory** - Remembers emotional context across conversations
- **🎭 Emotion Detection** - Analyzes emotions in text using transformers
- **📈 Pattern Tracking** - Identifies emotional trends over time
- **🔒 Privacy First** - Your data is secure and GDPR compliant

## 🚀 How to Use

1. **Analyze Emotion**: Enter your text and user ID, then click "Analyze Emotion"
2. **View History**: See your emotional patterns over time in the Memory History tab
3. **Clear Memory**: Reset your emotional history when needed

## 🏗️ Technology Stack

- **Cognee** - Hybrid graph-vector memory layer
- **Transformers** - HuggingFace emotion detection (j-hartmann/emotion-english-distilroberta-base)
- **Gradio** - Interactive web interface
- **Python** - Core implementation

## 🎓 The Problem: AI Amnesia

Traditional LLMs are stateless:
- ❌ No memory of past conversations
- ❌ Context window limits
- ❌ Can't learn from patterns
- ❌ Forgets emotional context

## 💡 The Solution: Cognee Memory

Cognee provides four core memory operations:

1. **Remember** - Store emotional interactions persistently
2. **Recall** - Retrieve relevant past contexts using semantic search
3. **Improve** - Build knowledge graph connections (cognify)
4. **Forget** - Remove data when needed (GDPR compliant)

## 📊 Use Cases

- 🏥 Mental health support with emotional history
- 🎓 Educational systems that adapt to student emotions
- 🛍️ Customer service that remembers past interactions
- 🎮 Gaming NPCs with emotional memory
- 👥 Social robots with persistent relationships
